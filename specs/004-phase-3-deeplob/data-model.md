# Data Model: Phase 3 — DeepLOB Page

**Branch**: `004-phase-3-deeplob` | **Date**: 2026-05-17

Four entities; all persistent state is files (Constitution IV).

---

## E1 — LOBDataset

Full FI-2010 benchmark. **Not committed to git** (too large); lives on
developer's `~/data_lake/fi2010/` and the Modal Volume.

**On-disk**: `${DEEP_FINANCE_DATA_DIR}/lob_fi2010.parquet`

**Fields**:

| Field | Type | Notes |
|---|---|---|
| `day` | `int8` | 1-10 (day index within FI-2010's 10-day window) |
| `stock_id` | `int8` | 1-5 (Nasdaq Nordic stocks; KESBV, OUT1V, SAMPO, RTRKS, WRT1V) |
| `tick` | `int32` | Sequential timestamp within (day, stock) |
| `bid_price_1..10` | `float32` | 10 bid price levels |
| `bid_volume_1..10` | `float32` | 10 bid volume levels |
| `ask_price_1..10` | `float32` | 10 ask price levels |
| `ask_volume_1..10` | `float32` | 10 ask volume levels |
| `label_k10` | `int8` | {0, 1, 2} = (down, stationary, up) at k=10 ticks |
| `label_k20, k30, k50, k100` | `int8` | Same shape at other horizons |

**Train/test split** (per FI-2010 convention):
- `day ∈ {1, ..., 7}` → train
- `day ∈ {8, 9, 10}` → test

**Row cardinality**: ~3.5M train rows + ~800k test rows ≈ 4.3M total.

## E2 — LOBDemoSlice

Tiny test-set subsample shipped in the repo for the live HF Space.

**On-disk**: `data/lob_fi2010_demo.parquet`

Same schema as E1. Selection: `(day == 8) AND (stock_id == 1)` → ~250k
rows. Size ≈ 10 MB compressed.

## E4 — LOBMetricsPanel

Per-method metrics + 3×3 confusion matrices, pre-computed offline.

**On-disk**: `data/backtests/lob_results.parquet`

**Fields**:

| Field | Type | Notes |
|---|---|---|
| `method` | `string` | `deeplob` / `mlp` / `cnn1` / `cnn2` / `lstm` / `lda` |
| `k` | `int8` | Always 10 in Phase 3 (other k's are paper-reported only) |
| `accuracy` | `float64` | macro accuracy |
| `precision_macro` | `float64` | |
| `recall_macro` | `float64` | |
| `f1_macro` | `float64` | |
| `cm_00` ... `cm_22` | `int64` | 9 cells of the 3×3 confusion matrix |
| `source` | `string` | `"reproduced_here"` or `"paper_reported"` |

**Row cardinality**: ≤ 30 rows (6 reproduced methods × 1 k +
~10 paper-reported entries from Table II at k=10).

**State transitions**: regenerated by `scripts/run_backtests.py --lob`.
